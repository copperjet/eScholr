## DEMO_SETUP — Manual steps after applying migrations

The demo seed (018) creates staff without `auth_user_id`. To log in for the board demo:

**Step 1 — Create auth users in Supabase Dashboard → Authentication → Users → Add User:**
```
admin@cis-demo.edu   / Demo@2026!
jkamau@cis-demo.edu  / Demo@2026!   (HRT — Joyce Kamau)
gtembo@cis-demo.edu  / Demo@2026!   (Finance — Grace Tembo)
```

**Step 2 — Link auth users to staff records (run in Supabase SQL Editor):**
```sql
UPDATE staff SET auth_user_id = (SELECT id FROM auth.users WHERE email = 'admin@cis-demo.edu')   WHERE email = 'admin@cis-demo.edu';
UPDATE staff SET auth_user_id = (SELECT id FROM auth.users WHERE email = 'jkamau@cis-demo.edu')  WHERE email = 'jkamau@cis-demo.edu';
UPDATE staff SET auth_user_id = (SELECT id FROM auth.users WHERE email = 'gtembo@cis-demo.edu')  WHERE email = 'gtembo@cis-demo.edu';
```

**Step 3 — Enable JWT hook:**
Dashboard → Auth → Hooks → Custom Access Token Hook → select `public.custom_access_token_hook`

**Step 4 — Set school code in app:**
App login screen: enter `CIS_DEMO` as school code.

After these steps, login with any demo credential. The JWT hook auto-injects `school_id`, `staff_id`, and `roles` into the token — all RLS policies and screen queries will work.
